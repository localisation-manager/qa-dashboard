# L10N QA Progress Dashboard

A live dashboard that shows testing-scenario progress per language for the **Fresha Partner** and **Marketplace** L10N QA databases in Notion.

- Static HTML + vanilla JS (no build step)
- Python script (`scripts/sync_notion.py`) pulls counts per language from the Notion API
- GitHub Actions workflow runs the sync daily, commits `data/data.json`, and republishes the site to GitHub Pages

## One-time setup (~15 minutes)

### 1. Create a Notion internal integration

1. Go to https://www.notion.so/profile/integrations (or Settings → Connections → Develop or manage integrations).
2. Click **New integration** → give it a name like `L10N QA Dashboard sync`.
3. Associated workspace: **fresha-app**.
4. Type: **Internal**.
5. Capabilities: you only need **Read content**.
6. Save → copy the **Internal Integration Token** (starts with `ntn_…`). Treat it like a password.

### 2. Share the two Notion databases with the integration

For each of the following databases, open it in Notion → click the `…` menu (top right) → **Connections** → add your new integration:

- **L10N Fresha Partner Testing Scenarios** (data source `4a2bfc1f-bbc8-4c20-831b-aadea0534cb6`)
- **L10N Marketplace testing scenarios** (data source `f1f72c71-4450-40b6-8492-e300d7caf69d`)

Without this step the script will get `object_not_found` errors.

### 3. Create the GitHub repo

1. Create a new repo on github.com (public or private — GitHub Pages requires a paid plan for private repos, so public is easier if the data isn't sensitive).
2. From this folder on your computer:

   ```bash
   cd l10n-dashboard
   git init
   git add .
   git commit -m "Initial dashboard scaffold"
   git branch -M main
   git remote add origin https://github.com/YOUR-USER/YOUR-REPO.git
   git push -u origin main
   ```

### 4. Add the Notion token as a GitHub secret

In the new repo on github.com:

1. **Settings → Secrets and variables → Actions → New repository secret**
2. Name: `NOTION_TOKEN`
3. Value: paste the `ntn_…` token from step 1.

### 5. Enable GitHub Pages

In the repo:

1. **Settings → Pages**
2. **Source**: `GitHub Actions`
3. Save.

### 6. Trigger the first sync

In the repo:

1. **Actions → "Sync Notion data & deploy dashboard" → Run workflow** (on `main`) → Run.
2. Wait ~1–2 minutes. On success, `data/data.json` will be committed and the dashboard deployed to `https://YOUR-USER.github.io/YOUR-REPO/`.

After that, the workflow runs itself every day at 07:00 UTC.

## Local preview

Just open `index.html` in a browser. The fetch for `data/data.json` will fail because of the `file://` protocol, and the dashboard will fall back to sample data with a warning banner.

To preview with the real data file locally, run any static server from this folder:

```bash
python -m http.server 8000
# then open http://localhost:8000
```

## How it works

- **`scripts/sync_notion.py`** calls the Notion `POST /v1/databases/{id}/query` endpoint for each of the two databases, paginates through all pages (100 at a time), and counts each language column's status values.
- **Status normalization**: `"Post launch"` and `"Post-launch"` are merged into `Post-launch`; `"Repeat 🔁"` and `"🔁 Repeat"` into `Repeat`; empty/missing values count as `Not started`.
- **`index.html`** fetches `data/data.json` on load and renders the leaderboard for each platform (Fresha Partner tab + Marketplace tab).

## Changing the sync schedule

Edit the `cron:` line in `.github/workflows/sync.yml`. Cron strings are evaluated in UTC.

| Desired time (CET, winter) | Cron |
| -- | -- |
| Every day at 08:00 | `0 7 * * *` |
| Every weekday at 09:00 | `0 8 * * 1-5` |
| Every 6 hours | `0 */6 * * *` |

## Adding or removing languages

Edit `PARTNER_LANGS` / `MARKETPLACE_LANGS` at the top of `scripts/sync_notion.py`. The column names must match Notion's property names exactly, including the trailing space on `"ID "`.

## Database IDs

If the databases are ever duplicated or the data-source IDs change, update `PARTNER_DB_ID` and `MARKETPLACE_DB_ID` in `scripts/sync_notion.py`.
